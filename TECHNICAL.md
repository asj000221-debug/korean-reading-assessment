# 기술 정리 — 한국어 난독증 읽기평가 시스템

> 구현 구조 / 평가 방식 / 모델 / 실험 결과 / 한계를 한 곳에 정리.

---

## 1. 문제 정의 (가장 중요)

이것은 **"음성 전사(transcription)"가 아니라 "정답 발음 대비 정합성 측정"** 문제다.
읽어야 할 단어/문장을 **이미 알고 있으므로**, "무슨 말을 하려 했나"가 아니라
**"이 정해진 음소열을 얼마나 정확히 읽었나"**를 측정한다.

핵심 설계 원칙:
- **Whisper 금지** — 디코더 LM이 오독을 자동 보정(self-correction)해 진단 신호를 지움.
  → 외부 LM 없이 음향 posterior로 디코딩하는 **CTC 계열(wav2vec2)** 만 사용.
- **학습 0** — 사전학습 모델을 추론만. G2P·정렬·채점은 전부 규칙 코드.
- **받침 일괄 무시 금지** — 문항 유형별로 채점 엄격도를 분기(난독증은 음운규칙
  적용 능력 = 비어휘 경로를 측정하므로).

---

## 2. 전체 파이프라인 (데이터 흐름)

```
                    ┌─ (트랙1) 한글 CTC: kresnik ──→ 한글 음절
[음성 16k mono] ─ASR┤
                    └─ (트랙2) 음소 CTC: slplab ───→ 음소열(로마자)
                                                          │
[정답 단어/문장] ─G2P(g2pkk)→ 발음(한글) ─┬─ 자모 분해(트랙1)
                                          └─ 음소 매핑(트랙2)
                                                          │
                          DP 정렬(단어/어절별 구간 배정) ←┘
                                          │
                          문항 유형별 채점 룰(조건문, 학습0)
                                          │
                    단어/어절별 정·오 + 오류유형 + 정확도 (+ 유창성 속도)
```

두 트랙(한글/음소)을 모두 구현해 비교 가능. **음소 트랙이 정밀도에서 우월**(§6).

---

## 3. 구성요소별 구현

### 3.1 ASR (음향 인식, CTC)
- **트랙1 (한글)**: `kresnik/wav2vec2-large-xlsr-korean` — 한글 음절 출력. `asr.py`
- **트랙2 (음소)**: `slplab/wav2vec2-xls-r-300m_phone-mfa_korean` — 45개 음소(로마자)
  출력. `phoneme_asr.py`
- 공통 전처리(`asr.preprocess`): **16kHz mono 강제** + 앞뒤 무음 트림 + 피크 정규화
  + 0.1s 패딩. 단어 단위 녹음의 인식 안정화.
- 디코딩: `argmax`(greedy CTC). 외부 LM 없음 → 오독을 고치지 않음.

### 3.2 G2P (정답 → 발음)
- `g2pkk`(g2pk 유지보수 포크). 철자 → 실제 발음. `g2p_expected.py`
- 예: 국물→궁물(비음화), 꽃을→꼬츨(연음), 해돋이→해도지(구개음화), 같이→가치.
- 문장은 전체를 G2P → **어절 넘는 연음까지 반영**(옛날 어느→옌나 러느), 공백으로
  어절 경계 보존. `pipeline_sentence.expected_eojeol_prons`
- 문항 유형별 '허용 발음 후보': transparent는 표면형도 허용, phonrule은 규칙형만.

### 3.3 음소 매핑 (트랙2 전용)
- `phoneme_map.py`: G2P 발음(한글) → slplab 음소(로마자). (자모,위치)→음소 테이블.
  - 초성 ㄱ→G, ㄲ→GG, ㅋ→Kh … / 종성 7종 중화 ㄱㄲㅋ→k, ㄷㅅㅈ…→t, ㅂㅍ→p,
    ㅇ→NG, ㄴ→N, ㄹ→L, ㅁ→M / 모음 ㅏ→A, ㅓ→EO, ㅗ→O, ㅜ→U, ㅡ→EU …
  - 검증: 궁물→`G U NG M U L`, 낟→`N A t` 등 모델 실측 출력과 정확히 일치.
- 역변환 `phones_to_hangul`: 음소열 → 한글 재조합(사용자에게 "들린대로" 표시용).

### 3.4 정렬 (alignment) — `align.py`
- **자모/음소 토큰**: 각 토큰 = `Jamo(char, role, syl)`. role=초성/중성/종성(음소
  모드는 미상). 한글은 (char,role)로, 음소는 char로 매칭(`key` 파라미터로 분기).
- **단어 1개**: `difflib.SequenceMatcher`로 자모 정렬 → 대치/생략/첨가 추출(`diff_tokens`).
- **목록/문장(연결발화)**: **DP 분할 정렬**(`align_tokens_dp`). 실제 발화 토큰열을
  단어 순서대로 '연속 구간'으로 최적 분할해 각 단어/어절에 배정.
  - 편집거리 비용 최소화. 누락된 단어는 빈 구간(→ 전부 오류)으로 처리.
  - **효과**: 1음절 단어가 누락돼 뒤 단어가 밀려도 각 단어에 맞는 구간을 찾아
    밀림을 흡수(단순 전역 정렬의 연쇄 오정렬 문제 해결).

### 3.5 채점 (비즈니스 로직) — `scoring.py`
- 입력: 단어, 문항 유형, 기대발음, 오류 목록. 전부 조건문(학습 0).
- 문항 유형별 룰:
  | 유형 | 룰 |
  |---|---|
  | `nonword` | 모든 음소/자모 차이 = 오류 (가장 엄격, 맥락 추측 불가) |
  | `phonrule` | 음운규칙 적용 실패 포함 모든 차이 = 오류 (규칙적용이 측정대상) |
  | `transparent` | 종성(받침) 단순 조음 오류는 가중치 0(관대), 단 '다른 실단어'가 되면 오류 |
- 정확도 = `1 - (가중오류 합 / 기대 단위 수)`, [0,1] 클램프. 분모는 자모수(트랙1)
  또는 음소수(트랙2).

### 3.6 유창성 속도 — `fluency.py`
- 분당 음절수(SPM) = 음절수 / 읽기시간(분). 읽기시간 = 발화 구간(첫~끝, 내부 멈춤 포함).
- 정확 음절수 기반 '분당 정확음절수'도 산출. (망설임/자기수정은 forced alignment
  필요 → v1)

### 3.7 인터페이스 — `webapp.py` (Gradio)
- 브라우저 마이크 녹음 → 서버(이 PC) 추론. `share=True`로 임시 공개 링크.
- 3개 탭: **목록 읽기**(해독), **문장 읽기**(유창성), **단어 1개**.
- 품질 게이트: 너무 작음(peak<0.05)/큼(클리핑)/짧음(<0.3s) → 재녹음 안내.
- 결과를 **"들린대로(한글)"** 로 표시해 사람이 판단 가능하게.

## 4. 평가(채점) 방식 — 무엇을 어떻게 판정하나

1. **단위**: 단어(목록) 또는 어절(문장)별로 정/오 + 정확도(%).
2. **오류 유형**: 대치/생략/첨가(+ 유형 태그 nonword_strict, phonrule_failure,
   coda_lenient_waived 등).
3. **집계**: 전체 정답 수 / 정확도 평균. 문장은 + 속도(SPM).
4. **검증(테스트)**: `tests/test_scoring.py` 유형별 정상/오독 단위테스트 10종.
   dry-run(텍스트/음소열 주입)으로 ASR 없이 정렬·채점 로직 점검.

## 5. 검사 모드 ↔ 실제 임상 대응

| 모드 | 실제 검사 | 측정 |
|---|---|---|
| 목록 읽기 | **해독(decoding) 검사** (KOLRA 해독) | 글자-소리 변환력(특히 무의미단어) |
| 문장 읽기 | **읽기 유창성** 검사 | 정확도 + 속도 |
| 단어 1개 | (참고용, 채점 단위만 동일) | — |

목록 읽기(무의미단어 포함)가 난독증 변별력 1순위.

## 6. 실험으로 확인한 핵심 사실

1. **모델·코드 정상**: kresnik는 깨끗한 문장(8~11초)을 거의 완벽 전사.
2. **짧은 단어 사각지대**: kresnik는 ~1초 미만 발화를 구조적으로 오인식(깨끗한
   데이터셋도 0.8초로 자르면 동일하게 깨짐). 한국어 단어 1개(0.5~0.8초)가 여기 빠짐.
3. **연결발화로 회피**: 여러 단어를 이어 읽으면(3초+) 인식이 크게 개선.
4. **음소 모델 우월**: slplab은 1음절 무의미단어(낟→`N A t`)까지 잡고, 음소 단위로
   더 충실(궁물·해도지·바다·구두·강 정확). 오류가 "못 알아들음"→"음소 1개 차이"로
   질적 전환. **읽기평가에 적합한 단위.**
5. **클리핑 주의**: peak≈1.0(과대 녹음)은 왜곡으로 인식 저하 → 품질 게이트로 차단.

## 7. 한계 & 실제 '진단'까지 필요한 것

지금 도구 = **해독/유창성 정확도 채점 엔진**. 난독증 '진단'은 추가로 필요:
- **또래 규준(norm)**: 정상 발달 아동 점수 분포 → 백분위/표준점수. (현재 원점수만)
- **음운처리 검사**: 음운인식(소리 조작)·작업기억·RAN — 글자 없는 별도 과제, 미구현.
- **배제진단**: 지능/감각/교육기회 — 범위 밖.
- **음소 미세차 보정**: 모음 근접쌍(ㅗ↔ㅜ) 등 ASR 변동을 GOP 신뢰도(STEP 5)로
  '불확실' 처리 후 사람 검수. **임상가 채점본과 임계값 캘리브레이션**(브리프 §8).
- **망설임/자기수정**: forced alignment 타임스탬프 기반(STEP 6 잔여).

## 8. 파일 구조

```
reading_assessment/
├── asr.py               ASR(한글 CTC) + 전처리 + 품질지표
├── phoneme_asr.py       ASR(음소 CTC, slplab)
├── g2p_expected.py      정답→발음(G2P), 유형별 허용발음
├── phoneme_map.py       한글발음↔음소 매핑/역매핑
├── align.py             자모/음소 분해·정렬(difflib + DP 분할)
├── scoring.py           문항 유형별 채점 룰
├── fluency.py           유창성 속도(분당 음절수)
├── pipeline.py          단어(개별) 파이프라인
├── pipeline_list.py     목록(한글) 파이프라인
├── pipeline_phoneme.py  목록(음소) 파이프라인
├── pipeline_sentence.py 문장(음소, 어절별 + 유창성)
├── webapp.py            Gradio 웹앱(3탭)
├── items.json           문항(단어 32 + 문장 6)
└── tests/test_scoring.py 유형별 단위테스트
```

## 9. 실행
```bash
pip install -r requirements.txt
python webapp.py            # 브라우저 마이크 + 임시 공개 링크
python tests/test_scoring.py
# dry-run(ASR 없이 채점 점검):
python pipeline_phoneme.py --dry "N A t Ph U p M EO L Th EU N ..."
python pipeline_sentence.py --sentence "해돋이를 보러 산에 같이 올라갔어요." --dry "<음소열>"
```
