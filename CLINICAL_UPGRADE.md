# 임상 업그레이드 — 채점기 → 진단·처방 엔진

> 익명화된 난독증 치료 사례연구(152회기 종단 기록)를
> 근거로, 기존 v0(해독 정확도 채점기)를 **음운인식·해독·쓰기 3영역 × 발달위계 ×
> 임상오류분류 × 종단추적 × 자동처방** 시스템으로 확장했다. 전부 규칙(학습 0) — 기존
> 설계 철학(Whisper 금지/CTC, 학습 0, G2P 발음 비교)을 그대로 유지한다.

---

## 1. 무엇이 부족했나 (PPT가 드러낸 격차)

PPT는 단순 단어 채점이 아니라, 한 임상가가 한 아동을 **3개 영역에서 / 발달 위계를
따라 / 오류의 질을 읽어 / 회기마다 추적하며 / 다음 목표를 처방**한 기록이다.

| 차원 | 기존 v0 | 임상 사례 현실 | 근거 |
|---|---|---|---|
| 측정 영역 | 해독 1개 | **음운인식·해독·쓰기 3개 병렬** | 5 (`[음운인식][해독][쓰기]`) |
| 오류 분석 | 대치/생략/첨가 | **임상 오류유형** (초성생략·도치·활음탈락·치찰음혼동…) | 19,25,32,45,50,65 |
| 구조 | 평면 3유형 | **발달 위계 DAG** (음절→음소, 단모음→이중모음, 첫자음→끝소리규칙→음운변동) | 8,109 |
| 추적 | 1회성 | **152회기 종단** (긍정/한계/개선방향) | 사례 전반 |
| 처방 | 없음 | 오류 → 다음 목표 추천 | 33,66,82,108 |

## 2. 무엇을 만들었나 (신규 모듈 5종 + 문항은행)

```
                       ┌─────────────────── 기존 v0 (유지) ───────────────────┐
[음성] → asr/phoneme_asr → g2p_expected → align → scoring  → 정확도%
                                                     │
                       └─────────────────────────────┼────────────────────────┘
                                                     ▼   신규 임상 계층
                              error_taxonomy.py  ── 오류의 *질* 분류(난독 신호)
                                                     │
                              skill_map.py       ── 발달 배치 + 다음목표 + 처방
                                                     │
                              phon_awareness.py  ── 음운인식(글자없는 청각과제)
                                                     │
                              learner_profile.py         ── 회기간 종단 추적
                                                     │
                              diagnose.py        ── 위를 결합한 임상 리포트
                              item_bank.json     ── PPT 실자극어 레벨별 문항은행
```

### 2.1 `error_taxonomy.py` — 임상 오류유형 분류기
대치/생략/첨가라는 *형태*가 아니라, 난독증 *진단 신호*로 분류한다. 조음·음운 **자질표**
(조음위치/방법/발성유형/치찰성, 모음 활음/기본모음)를 적용해 14종 임상유형 + 약점
하위기술 + 심각도를 산출. **PPT 실관찰 오류 9종이 모두 정확히 분류됨**:

| 사례 관찰 | 분류 결과 | 근거 |
|---|---|---|
| 가죽→아욱 | `onset_deletion` [high] (무음초성 ㅇ 처리) | 50 |
| 가방→바강 | `syllable_transposition` [high] | 25 |
| 자두→사두, 진달래→신달래 | `sibilant_confusion` [high] (ㅅ↔ㅈ) | 32 |
| 야→아 | `glide_simplification` [high] | 25,45 |
| 카→콰 | `glide_insertion` [med] | 50 |
| 웨이터→워이터 | `w_diphthong_confusion` [high] | 32 |
| 눈꺼풀→눈떠풀 | `place/phonation_confusion` [med] | 65 |
| 멀튼→머튼 | `coda_deletion` [low] | 25 |
| 국물→국물 | `rule_failure:비음화` [high] | 90-102 |

> 설계 포인트: 초성 `ㅇ`은 음가 없는 무음이므로 `자음→ㅇ` 대치는 **초성 생략**(난독
> 핵심 신호)으로, `ㅇ→자음`은 **초성 첨가**로 재해석한다. difflib이 onset 자리 대치로
> 보는 케이스를 임상적으로 올바르게 잡는다.

### 2.2 `skill_map.py` — 발달 위계 + 배치/처방 엔진
사례연구 치료절차(사례 관찰/109)를 **음운인식·해독·쓰기 27노드 DAG**로 인코딩. 각 노드 =
{영역, 발달순서(회기), 전제조건, 근거사례 관찰 **사례의 실제 중재법**}.
- `placement(evidence)` — 검사 증거로 *현재 막힌 단계* + *전제조건 충족된 다음 목표* 산출
- `prescribe(weak_skills)` — 약점을 **발달 낮은 것부터** 정렬해 개선방향(중재법) 처방.
  중재법은 사례의 실제 방법 근거: 입모양 사진(사례 관찰), 자석판(19), Slow 합성(39),
  모음카드 Quick Drill(29) 등.

### 2.3 `phon_awareness.py` — 음운인식 과제 모듈 (빠진 영역 보강)
글자 없이 소리만 조작하는 과제(사례 관찰). **기대 응답 = 입력의 변환**이라는
통찰로 기존 ASR+정렬 코어를 재사용. 음절/음소 수준 7연산(수세기·합성·분절·생략·첨가·
대치·도치) 생성 + 채점. 도치 과제는 사례 핵심약점(가방→바강)을 직접 측정.

### 2.4 `learner_profile.py` — 종단 학습자 프로파일
회기 결과를 누적(JSON)하고 스킬 숙달도·오류빈도·유창성(어절/분) 추이를 추적.
임상가가 손으로 쓰던 **학습결과(긍정/한계) + 개선방향을 자동 생성**(사례 관찰…
의 형식 그대로). 규준(norm) 대조는 자리만 확보(규준표 수급 시 백분위 산출).

### 2.5 `diagnose.py` — 진단·처방 오케스트레이터
한 회기 채점 결과 → 오류 프로파일 → 약점 스킬 → 발달 배치 → 개선방향 → 종단 누적의
전 과정을 결합해 **임상 리포트** 출력(`python diagnose.py`로 시연).

### 2.6 `item_bank.json` — PPT 실자극어 문항은행
기존 32개 → PPT의 실제 레벨별 자극어 수백 개를 15개 스킬노드 + 문장 + 표기인식쌍 +
철자오류규준으로 구조화. 청각변별/오류극복은 최소대립쌍(target|distractor)으로.

## 3. 검증

```bash
python tests/test_scoring.py    # 기존 채점 10/10
```

## 4. 설계상 특징

1. **채점(score) → 진단(diagnose) → 처방(prescribe)**: "75%"가 아니라 "활음 단순화 →
   미끄러지는 모음 단계 → /이.아/ 합성 연습"을 출력한다.
2. **학습 0 유지**: 파인튜닝·임계값 캘리브레이션 없이, 음운학 자질표 + 발달위계 규칙만으로
   임상 신호를 추출. 기존 v0의 설계 철학을 그대로 이어간다.
3. **실데이터 정초(grounding)**: 오류유형·중재법·문항을 실제 임상기록(PPT)에서 도출 —
   한 언어재활사(SLP)의 152회기 기록을 코드화했다.
4. **종단 설계**: 난독 중재의 핵심인 시간에 따른 변화를 1급 객체로 다룬다.
