📖 한국어 난독증 아동 읽기평가 시스템 (v0)

구성·구현·평가 정리 · 학습 0(사전학습 모델 추론 + 규칙 코드)
CTC 음소인식G2P DP 정렬유형별 채점 GOP 신뢰도유창성 Gradio

정해진 단어·문장을 소리 내어 읽은 음성을 정답 발음과 비교해 읽기 정확도를 자동 채점한다. 핵심 가치는 평가자 의존성 제거(일관된 자동 채점).

1. 핵심 아이디어 — "전사"가 아니라 "정답 대비 채점"

읽어야 할 내용을 이미 알고 있으므로, "무슨 말을 하려 했나"(자유 전사)가 아니라 "이 정해진 음소열을 얼마나 정확히 읽었나"를 측정한다.

원칙이유
Whisper 안 씀 디코더 언어모델이 오독을 자동 보정([채상]→책상)해 진단 신호를 숨김 → 소리에 충실한 CTC(wav2vec2)만 사용
학습 0 공개 사전학습 모델을 추론만. G2P·정렬·채점은 일반 파이썬 코드
받침 일괄 무시 금지 음운규칙 적용력 = 난독증 핵심 신호 → 문항 유형별로 채점 엄격도 분기

2. 전체 처리 흐름

[음성 16kHz mono] │ ▼ ① 음소 인식 (CTC, slplab) 음소열 예) "G U NG M U L" │ [정답 단어/문장] │ │ ② G2P (g2pkk): 국물 → 궁물 │ ▼ + 음소 매핑 → 기대 음소열 │ "G U NG M U L" ▼ ▼ ③ DP 정렬 ── 단어/어절별로 실제 음소를 기대 음소에 최적 배정 │ ▼ ④ 채점: 문항 유형별 규칙 + GOP(발음 신뢰도) ▼ 단어/어절별 정·오 / 정확도 / (문장은) 읽기 속도

3. 단계별 구성요소

① 음향 인식 (ASR · CTC)

  • 음소 모델(주력) slplab/wav2vec2-xls-r-300m_phone-mfa_korean — 한국어 음소 45개를 로마자로 직접 출력. 1음절·짧은 단어에 강함.
  • 한글 모델(비교) kresnik/wav2vec2-large-xlsr-korean — 한글 음절 출력.
  • 오디오는 무조건 16kHz mono로 변환 + 앞뒤 무음 트림·정규화(단어 인식 안정화).
  • 외부 언어모델 없는 argmax 디코딩 → 오독을 고치지 않음. asr.py / phoneme_asr.py

② G2P (정답 → 기대 발음)

  • g2pkk로 철자를 실제 발음으로: 국물→[궁물](비음화), 꽃을→[꼬츨](연음), 해돋이→[해도지](구개음화).
  • 문장은 전체를 G2P → 어절 넘는 연음까지 반영(옛날 어느→옌나 러느), 공백으로 어절 분리.
  • 배포 환경(mecab 없음) 대비 고정 문항 발음을 미리 계산해 캐시(pron_cache.json). g2p_expected.py / phoneme_map.py

③ 정렬 (alignment)

  • 단어 1개: difflib로 자모 정렬 → 대치/생략/첨가 추출.
  • 목록·문장(연결발화): DP 분할 정렬 — 실제 음소열을 단어/어절 순서대로 '연속 구간'으로 최적 분할해 배정. 1음절 단어가 누락돼 뒤가 밀려도 흡수. align.py

④ 채점 (문항 유형별 규칙)

유형측정채점 엄격도
무의미단어 (낟·풉·멀튼)순수 해독력(맥락 추측 불가) 가장 엄격 — 거의 모든 음소 차이를 오류로
음운규칙 단어 (국물·꽃을)음운규칙 적용 능력 규칙 적용 실패를 오류로
자소-음소 일치 (바다·구두)기본 자모 대응 받침 단순 조음 오류는 관대(단, 다른 실단어 되면 오류)

scoring.py · 정확도 = 1 − (가중오류 / 기대 음소 수)

⑤ GOP — "제대로 읽었는데 틀림" 줄이기

모델이 시→지처럼 비슷한 소리로 잘못 출력해도, 기대 음소를 음향에 강제정렬해 "그 소리를 실제로 냈는지"의 신뢰도(GOP)를 직접 측정해 판정한다.

  • GOP 높음 = 또렷이 발음함 → 정답(자유인식이 틀려도 구제)
  • GOP 낮음 = 그 소리를 못 들음 → ⚠ 확인필요(오독일 수도, 인식 한계일 수도)

torchaudio forced_align 사용. gop.py · 임계값은 잠정값(임상가 채점본으로 보정 필요).

⑥ 유창성 (속도)

문장 읽기에 분당 음절수(SPM) 추가 — 정확도 + 속도로 실제 '읽기 유창성' 검사에 근접. fluency.py

6. 구축 과정에서 해결한 문제들

짧은 단어 사각지대
CTC 모델은 ~1초 미만 발화를 구조적으로 오인식(깨끗한 데이터도 0.8초로 자르면 깨짐). → 여러 단어를 이어 읽는 '목록/문장' 모드로 회피.
모델 정밀도
kresnik는 단어를 대충 맞힘 → 음소 모델(slplab)로 전환해 음소 단위 정밀도 확보 (낟→N A t 등 1음절도 포착).
제대로 읽었는데 오답
ASR 미세 오인식(시↔지) → GOP 주채점으로 기대 음소 실현 신뢰도 직접 판정.
결과 가독성
로마자 음소(N A k…)는 못 읽음 → 음소→한글 복원으로 "들린대로" 표시.

7. 성능 평가 (냉정하게)

항목성능근거(실측)
연결발화 2+음절 단어🟢 양호 멀튼·궁물·해도지·바다·구두·강 음소 단위로 잘 인식
1음절 무의미단어(낟/풉)🔴 불안정 누락·받침 흔들림 — 진단상 가장 중요한데 가장 약함
음소 미세차(ㅗ↔ㅜ, 시↔지)🟡 애매 ASR 변동. GOP로 일부 구제, 임계값은 잠정
문장 어절 정확도🟢 양호연음 반영, 어절별 정/오
파이프라인 안정성🟢 완성정렬·채점·GOP·유창성 구조 견고, 단위테스트 통과
한 줄 평가: "동작하는 v0 개념증명"으로는 성공. 하지만 "임상에서 바로 믿고 쓸 정밀도"는 아직 아님 — off-the-shelf 모델의 천장에 막혀 있고, 정확도 도약은 파인튜닝 + 캘리브레이션(데이터) 단계가 필요.

8. 한계 & 다음 단계

  • 또래 규준 없음 → 백분위 산출 불가(현재 원점수만). 진단엔 필수.
  • 음운처리 검사(음운인식·작업기억·RAN) 미구현 — 글자 없는 별도 과제.
  • 파인튜닝: ① 아동 정상음성(자동 라벨, 지금 가능) → ② 임상 라벨(SLP 전사, 보정·규준 동시 해결). 주의: 라벨은 '실제 발음한 음소'로 — 정답 텍스트로 학습하면 self-correction 발생.
  • 캘리브레이션: GOP 임계값을 임상가 채점본과 대조(1순위 검증=무의미단어 일치도).
